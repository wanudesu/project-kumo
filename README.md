# 🇯🇵 Japanese

## プロジェクト紹介

**KUMO**は、日本（東京・大阪）の求人情報を地図上で探し、  
求人者と求職者がリアルタイムでやり取りできる**地図ベースのマッチングプラットフォーム**です。  
韓国語・日本語の同時対応、5人チームで制作しました。

> 🏆 SCIT プロジェクト発表 奨励賞（3位）受賞

---

## 主要機能

### 🗺️ スマートマップ検索
- Google Maps連携＆クラスタリングで大量の求人を地図上に表示
- GPS基盤の現在地周辺フィルタリング（東京・大阪対応）
- ボトムシート＆フローティングカードで移動中も求人確認

### 💬 リアルタイム1:1チャット
- STOMP / WebSocketベースで求人者・求職者間のチャットルームを即時生成

### 🌐 日韓多言語対応
- ユーザー設定に応じてKR/JP UIと求人内容を自動切替
- DeepL API連携

### 💼 求人管理
- 求人者専用ダッシュボード、求人の登録・修正・削除
- 応募者管理・スカウト機能
- 悪質求人の通報システム

### 🌙 ダークモード対応
- ダーク/ライトモード切替とアンチフリッカー処理

## 私が担当したこと

このリポジトリはチームプロジェクト全体ではなく、  
**私が直接担当したリクルーター（求人者）ページの実装とUI/UX設計**を中心に整理した個人フォークです。

### 画面設計（Figma）
- リクルーター専用画面の全体フローをFigmaで設計してから実装
- 管理者ページを除く主要画面のUI/UX方向の提案・調整

### フロントエンド実装
| 画面 | 内容 |
|------|------|
| メインダッシュボード | リクルーター向けトップ画面、主要機能へのナビ構成 |
| 求人一覧 / 詳細 | 登録求人の確認・修正・削除 |
| 求人登録 / 修正 | 入力フォーム設計、会社情報連動で住所自動入力 |
| 会社登録 | 企業情報登録と求人登録へのスムーズな連携設計 |
| 求人者マイページ | 担当者情報の確認・修正画面の実装 |
| ミニカレンダーサイドバー | どの画面からも予定確認できるright sidebar実装 |

### その他
- ダークモードの色調整と**テーマ切替時のちらつき現象を発見・修正**
- 全体デザインのトーン統一と使いやすさの改善提案

---

## 工夫した点

**入力フローの整理**  
求人登録フォームは入力項目が多くなりがちなため、Figmaで画面フローを先に整理してから実装しました。

**会社情報との連動**  
登録済みの会社情報に応じて、求人登録時の住所が自動で切り替わるよう構成し、入力の手間を減らしながら整合性も保ちました。

**ミニカレンダーの常時表示**  
求人管理・登録作業中でもすぐ予定を確認できるよう、right sidebarにミニカレンダーを固定してメインカレンダーと連動させました。

# 🇰🇷 Korean

## 프로젝트 소개

**KUMO**는 일본(도쿄·오사카) 지역의 구인·구직 정보를 지도 위에서 탐색하고,  
구인자와 구직자가 실시간으로 소통할 수 있는 **지도 기반 매칭 플랫폼**입니다.  
한국어와 일본어를 동시에 지원하며, 5인 팀 프로젝트로 제작되었습니다.

> 🏆 SCIT 프로젝트 발표 장려상 (3위) 수상

---

## 주요 기능

### 🗺️ 스마트 맵 탐색
- Google Maps 연동 및 마커 클러스터링으로 수백 개의 공고를 지도 위에 렌더링
- GPS 기반 현재 위치 반경 내 공고 필터링 (도쿄·오사카 지원)
- 바텀시트 & 플로팅 카드로 지도 이동 중 공고 요약 확인

### 💬 실시간 1:1 채팅
- STOMP / WebSocket 기반으로 구인자-구직자 간 즉각적인 채팅방 생성

### 🌐 한·일 다국어 지원
- 사용자 설정에 따라 한국어/일본어 UI 및 공고 내용 자동 전환
- DeepL API 연동

### 💼 공고 관리
- 구인자 전용 대시보드, 공고 등록·수정·삭제
- 지원자 관리 및 스카우트 기능
- 악성 공고 신고 시스템

### 🌙 다크모드 지원
- 다크/라이트 모드 전환 및 플리커 방지 처리

---

## 내가 담당한 부분

이 저장소는 팀 프로젝트 전체가 아니라,  
**제가 직접 담당한 리쿠르터(구인자) 페이지의 구현과 UI/UX 설계**를 중심으로 정리한 개인 포크입니다.

### 화면 설계 (Figma)
- 리쿠르터 전용 화면 전체 흐름을 Figma로 먼저 설계한 뒤 구현
- 어드민 페이지를 제외한 주요 화면의 UI/UX 방향 제안 및 조정

### 프론트엔드 구현
| 화면 | 내용 |
|------|------|
| 메인 대시보드 | 리쿠르터 진입 화면, 주요 기능 바로가기 구성 |
| 공고 목록 / 상세 | 등록 공고 확인, 수정·삭제 |
| 공고 등록 / 수정 | 입력 폼 설계, 회사 정보 연동으로 주소 자동 입력 |
| 회사 등록 | 기업 정보 등록 및 공고 등록과 연결되는 흐름 설계 |
| 구인자 마이페이지 | 담당자 정보 확인·수정 화면 구현 |
| 미니캘린더 사이드바 | 어느 화면에서도 일정 확인 가능한 right sidebar 구현 |

### 기타
- 다크모드 색감 조정 및 **테마 전환 시 플리커 현상 발견 및 수정**
- 전체 디자인 톤 통일 및 사용성 개선 제안

---

## 신경 쓴 점

**입력 흐름 정리**  
공고 등록 화면은 입력 항목이 많아질 수 있어서, Figma로 화면 흐름을 먼저 정리한 뒤 구현했습니다.

**회사 정보 연동**  
기존에 등록된 회사 정보에 따라 공고 등록 시 주소가 자동으로 바뀌도록 해, 입력 부담을 줄이면서 일관성도 유지했습니다.

**미니캘린더 상시 노출**  
공고 관리나 등록 작업 중에도 일정을 바로 확인할 수 있도록, right sidebar에 미니캘린더를 고정하고 메인 캘린더와 연동했습니다.
