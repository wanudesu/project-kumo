# 🇯🇵 Japanese

## 概要

KUMOは、日韓向けの求人・求職情報を地図ベースで確認できるマッチングプラットフォームです。  
Google Maps APIを活用し、位置情報をもとに求人情報を視覚的に確認できるように設計しました。

このリポジトリは、チームプロジェクト全体ではなく、**その中で私が担当したリクルーター向け機能の実装・画面設計・UI/UX調整**を中心に整理した個人用フォークです。

## サービスの主な特徴

- 地図ベースで求人情報を確認できる検索体験
- 求人者・求職者それぞれに合わせた機能構成
- 日本語・韓国語の多言語対応
- ダークモード対応による視認性向上

## 私が担当したこと

- リクルーター向けページ全体の画面設計
- Figmaを用いたUI設計と画面フロー整理
- リクルーター向け主要画面のフロントエンド実装
- リクルーター機能に関するCRUD処理およびDB連携
- ダークモード表示の調整とアンチフリッカー対応
- 画面全体のデザイン方向と使いやすさの改善提案

## 担当範囲

### 1. メインダッシュボード
- リクルーター向けトップ画面のレイアウト設計
- スケジュールや主要機能へすぐ移動できる構成に調整

### 2. カレンダー / ミニカレンダー連動サイドバー
- 日程をすぐ確認できるよう、各画面から見える位置にミニカレンダーを配置
- メインカレンダーと right sidebar を連動させ、確認のしやすさを意識

### 3. 求人管理
- 登録済み求人の一覧表示
- 求人情報の確認・修正・削除機能
- 管理画面として見やすい情報配置を意識して構成

### 4. 求人登録 / 求人修正
- 入力フォーム画面の設計と実装
- 登録済み会社情報に応じて住所が自動で切り替わるように構成
- 入力項目の流れが複雑になりすぎないよう整理

### 5. 会社登録
- 企業情報を登録・管理する画面の設計と実装
- 後続の求人登録機能とつながるように画面構成を調整

### 6. 採用担当者マイページ
- 採用担当者が自分の情報を確認・修正できる画面を実装
- 全体のデザインガイドラインに合わせて統一感を調整

※ 求職者管理・スカウト機能は担当範囲外です。

## 工夫した点

### ミニカレンダーを常に確認できる構成
求人管理や登録作業の途中でも日程をすぐ確認できるように、  
**ミニカレンダーを right sidebar に固定し、複数画面で一貫して使える形**にしました。

### 入力フローのわかりやすさ
求人登録画面では、入力項目が多くなりやすいため、  
**画面の流れと情報のまとまりを意識して、先にFigmaで整理してから実装**しました。

### 会社情報との連動
すでに登録されている会社情報に応じて、求人登録時の住所が自動で切り替わるようにし、  
**入力の手間を減らしながら整合性も保てるように構成**しました。

### ダークモードの見え方調整
単にダークモードを適用するだけでなく、  
**画面ごとの見やすさや色のバランス、切り替え時の違和感**も確認しながら調整しました。

# 🇰🇷 Korean

## 개요

KUMO는 한일 구인·구직 정보를 지도 기반으로 확인할 수 있는 매칭 플랫폼입니다.  
Google Maps API를 활용해 위치 정보를 시각적으로 확인할 수 있도록 구성했습니다.

이 저장소는 팀 프로젝트 전체 소개가 아니라, **그 안에서 제가 담당한 리쿠르터 페이지 구현, 화면 설계, UI/UX 조정**을 중심으로 정리한 개인용 포크 레포입니다.

## 서비스 주요 특징

- 지도 기반으로 공고를 탐색할 수 있는 구조
- 구인자 / 구직자에 맞춘 기능 구성
- 한국어 / 일본어 다국어 대응
- 다크모드 지원을 통한 가독성 향상

## 제가 담당한 부분

- 리쿠르터 페이지 전체 화면 설계
- Figma를 활용한 UI 설계 및 화면 흐름 정리
- 리쿠르터 주요 화면 프론트엔드 구현
- 리쿠르터 기능 관련 CRUD 처리 및 DB 연동
- 다크모드 색감 조정 및 안티 플리커 대응
- 전체적인 디자인 방향과 사용성 개선 제안

## 담당 범위

### 1. 메인 대시보드
- 리쿠르터용 메인 화면 레이아웃 설계
- 일정과 주요 기능으로 바로 이동할 수 있도록 구성

### 2. 캘린더 / 미니캘린더 연동 사이드바
- 어느 화면에서든 일정을 빠르게 확인할 수 있도록 미니캘린더 배치
- 메인 캘린더와 right sidebar를 연동해 확인성을 높임

### 3. 공고 관리
- 등록된 공고 목록 확인
- 공고 정보 수정 / 삭제 기능
- 관리 화면으로서 정보가 한눈에 보이도록 구성

### 4. 공고 등록 / 공고 수정
- 입력 폼 화면 설계 및 구현
- 등록된 회사 정보에 따라 주소가 자동으로 바뀌도록 구성
- 입력 흐름이 복잡해지지 않도록 화면 구조 정리

### 5. 회사 등록
- 회사 정보 등록 및 관리 화면 설계 / 구현
- 이후 공고 등록과 자연스럽게 연결되도록 구성

### 6. 구인자 마이페이지
- 구인 담당자가 자신의 정보를 확인 / 수정할 수 있는 화면 구현
- 전체 디자인 톤에 맞게 통일감 조정

※ 구직자 관리, 스카우트 기능은 담당 범위에서 제외됩니다.

## 신경 쓴 점

### 미니캘린더를 항상 볼 수 있는 구조
공고 관리나 등록 작업 중에도 일정을 바로 확인할 수 있도록  
**right sidebar에 미니캘린더를 고정하고 여러 화면에서 일관되게 보이도록 구성**했습니다.

### 입력 흐름 정리
공고 등록 화면은 입력 항목이 많아질 수 있어서,  
**화면 흐름과 정보 묶음을 먼저 Figma로 정리한 뒤 구현**했습니다.

### 회사 정보 연동
기존에 등록된 회사 정보에 따라 공고 등록 시 주소가 자동으로 바뀌도록 해,  
**입력 부담을 줄이면서 정보 일관성도 유지할 수 있게 구성**했습니다.

### 다크모드 조정
단순히 다크모드를 넣는 데서 끝나지 않고,  
**화면별 가독성, 색 균형, 전환 시 위화감**까지 확인하며 조정했습니다.
